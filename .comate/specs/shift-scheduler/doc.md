# 排班页面设计文档

## 一、需求概述

### 业务背景

为"选择困难症诊疗室"小程序新增「值班怎么排？」功能页，根据首页卡片入口（pagePath 已留空）跳转至排班页面。

### 排班规则

| 规则 | 说明 |
|------|------|
| 工作日 (周一/三/五) | 所有人无特殊情况均上班 |
| 轮休日 (周二/四/六/日) | 可轮休，但至少保证一人在岗 |
| 每日备班 | 每天安排备班人员（护士长不参与备班/夜班，休息日无备班） |
| 备班后休 | 晚上备班后第二天需休息；或头天下午无手术则备班人员可不来 |

### 人员构成

| 类型 | 人数 | 班型 | 备注 |
|------|------|------|------|
| 护士长 | 1 | 白班 | 不参与备班和夜班 |
| 倒班人员 | 3 | 主→夜→下（循环） | 三人错位排列，每日保证主夜各一人 |
| 白班人员 | 3 | 白班 + 备班轮换 | 每周各轮一次备班 |
| 备孕人员 | 1 | 白班 | 不参与备班 |

**班次类型**：主班(白天主班)、夜班、休息(下班)、备班、白班

### 人员定义

```js
staff: [
  { id: 'head',   name: '护士长', role: 'head',     color: '#F16B59' },
  { id: 'r_a',    name: '倒班A', role: 'rotate',   color: '#8E97FD', offset: 0 },
  { id: 'r_b',    name: '倒班B', role: 'rotate',   color: '#8E97FD', offset: 2 },
  { id: 'r_c',    name: '倒班C', role: 'rotate',   color: '#8E97FD', offset: 1 },
  { id: 'd_a',    name: '白班A', role: 'day',      color: '#FEB18F', standbyOffset: 0 },
  { id: 'd_b',    name: '白班B', role: 'day',      color: '#FEB18F', standbyOffset: 1 },
  { id: 'd_c',    name: '白班C', role: 'day',      color: '#FEB18F', standbyOffset: 2 },
  { id: 'd_prep', name: '白班D', role: 'day_prep', color: '#F3E6CE' },
```

### 排班生成算法

**倒班人员**（三人循环：主→夜→下）
```js
const rotateCycle = ['主', '夜', '下']
// 以某固定基准日期计算偏移
shift = rotateCycle[(dayIndex + offset) % 3]
```

**白班人员规则**
- 周一/三/五：全部上白班
- 周二/四/六/日（轮休日）：
  - 至少保证1人上班
  - 备孕者按轮休安排休息（轮休日每两周休息1次，保证不全休）
  - 3名白班护士轮流安排备班（每人每周备班1次）
  - 备班在非休息日的工作日晚上

**护士长规则**
- 周一/三/五：白班（标记为"长"）
- 周二/四/六/日：参与轮休排班，但不参加备班和夜班

## 五、交互设计

1. **月份切换**：左右箭头切换月份，重新生成该月排班
2. **周次 Tab**：切换当前查看的周，表格只展示当周 7 天
3. **单元格点击**：进入"编辑"状态，弹出班次选择器（Picker），支持手动修改某人某天的班次
4. **长按单元格**：弹出快捷操作（设为休息 / 设为备班）
5. **规则卡片折叠展开**：底部规则说明区域可折叠

---

## 六、边界条件与异常处理

| 场景 | 处理方式 |
|------|----------|
| 轮休日所有白班人员恰好全休 | 算法兜底：强制保留至少一人上白班 |
| 倒班人员病假/调班 | 手动编辑覆盖自动排班 |
| 月末跨周 | 按自然周处理，不足7天的周正常展示 |
| 人员名字过长 | 截断为4个汉字，tooltip展示全名 |
