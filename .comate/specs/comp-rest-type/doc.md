# 新增「休(补)」班次类型并修复备班块时序

## 需求概述

1. **新班次类型 `休(补)`**：备班后全天补偿休息（上午补休 + 下午休息），计入 T_week 0.5 天
2. **麦苗周日应为 `休(补)`**：`补` 类型备班（兜底）执行后也应生成次日 `休(补)`
3. **两个白班休息不足**：块分配时序错误导致，`_setAMRest` 应在块完成后才触发

---

| 字段 | 值 |
|---|---|
| 类型标识 | `'休(补)'` |

| 文字色 | `#7B87C8` |

| 含义 | 备班后全天补偿（上午补休 + 下午休息）|

## 根因分析与修复方案

### 问题 1：补类型备班不生成次日 `休(补)`（麦苗周日问题）

当前代码：
```javascript
if (cell.type === '白') {
  _setAMRest(...)   // ← 仅白班触发
}
// 补类型：什么都不做 → 周日无休(补)
```

修复：`'休(补)'` 类型（兜底备班）也应调用 `_setAMRest`：
```javascript
if (cell.type === '白') { ... }
else if (cell.type === '休(补)') {
  _setAMRest(schedule, date, staffId)  // 生成次日 休(补)
}
```

---

### 问题 2：块时序错误（白班休息不足根因）

`_setAMRest` 在块**第一天**就触发，导致：
```
Day N:   白+备 → 立即生成 Day N+1=休(补)
Day N+1: 休(补)（被排出 pass=0）→ rotate 截走备班 → 块断裂
```

修复：`_setAMRest` 仅在**块完成时**（consecBefore + 1 >= maxConsecutive）触发：
```javascript
if (cell.type === '白') {
  // 每个块日：转换他人为上半
  for (...) otherCell.type = '上半'
  // 仅块最后一天：生成次日 休(补)
  if (consecBefore + 1 >= maxConsecutive) {
    _setAMRest(schedule, date, staffId)
  }
} else if (cell.type === '休(补)') {
  _setAMRest(schedule, date, staffId)
}
```
