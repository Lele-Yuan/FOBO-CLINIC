# 排班系统重构 — 完成总结

## 解决的问题

1. **跨月排班不一致**：根本原因是备班指针按月重置，导致同一周从不同月份视角生成结果不同。通过引入全局时间线（从 `SCHEDULE_ANCHOR = 2026-04-06` 出发，备班指针 pointer=0 连续正向模拟），彻底消除此问题。

2. **主班/备班约束未满足**：Pass2 排除条件补加 `'主'`，防止主班人员被抢占为备班。兜底逻辑改为选夜班人员（不破坏主班）。

3. **查看范围受限**：移除月导航 + 周 Tab 架构，改为纯周导航（`prevWeek`/`nextWeek`），无上限向前/向后翻周。

4. **过去日期可编辑**：`onShiftTap` 前置日期检查，过去日期弹出 toast 并拦截。

5. **`午` 班次混乱**：彻底移除 `午` key，统一为：
   - `上` = 上午休（备班次日，半截渐变：上灰下白）
   - `下` = 倒班全天休（纯色）/ 下午休（白班人员，半截渐变：上白下灰）

6. **白班次日补午规则取消**：Pass3 整体删除。

## 核心架构变化

| 项目 | 重构前 | 重构后 |
|------|--------|--------|
| 生成单元 | 按月独立生成 | 全局时间线（锚点到 +400 天） |
| 备班指针 | 每月重置 | 连续不重置 |
| 导航方式 | 月导航 + 周 Tab | 纯周导航 |
| 半天班次 | `上`/`午` 两 key | `上`（上午休）/ `下`（下午休，白班人员）|
| Pass 数量 | 4 pass | 3 pass（Pass3 已移除）|
| 过去日期 | 可编辑 | 拦截 + 灰化显示 |

## 修改文件

- `pages/schedule/schedule.js`：完整重写（全局时间线算法 + 周导航逻辑）
- `pages/schedule/schedule.wxml`：周导航替换月导航 + 动态渐变渲染 + isPast 样式
- `pages/schedule/schedule.wxss`：新增过去日期灰化 + 移除旧 Tab 样式 + overflow:hidden 支持渐变圆角
