# 新增休(补)班次 — 实施总结

## 完成内容

所有修改集中在 `pages/schedule/schedule.js` 单文件，共 5 处改动。

### 1. SHIFT_COLORS（LINE 23）
新增 `'休(补)'` 颜色定义：
```js
'休(补)': { bg: '#D8E8F0', text: '#6A8FA0' }
```
使用带蓝调浅灰，与普通 `'休'`（#E8EAF0）视觉可区分。

### 2. LEGEND_LIST（LINE 35）
在图例末尾追加条目：
```js
{ label: '休(补)', desc: '补休全天（计0.5天，仅2人白班池备班次日）', bg: '#D8E8F0', text: '#6A8FA0' }
```

### 3. _calcRestScore（LINE 339）
```js
if (t === '休(补)') score += 0.5
```
`'休(补)'` 计入周配额 0.5 天；`'补'` 保持 0 不变。

### 4. _setAMRest（LINE 218-229）
新增 `dayPoolSize` 参数，根据白班池大小选择补偿类型：
- `dayPoolSize === 2` → `'休(补)'`（全天休，0.5 天配额）
- `dayPoolSize >= 3` → `'补'`（上午休/下午上班，0 配额，逻辑不变）

防重覆保护列表同步更新：`['休', '下', '上半', '休(补)']`。

### 5. pass2Standby（LINE 162）
```js
const dayPoolSize = staffList.filter(p => p.type === 'day').length
```
在调用 `_setAMRest` 时传入 `dayPoolSize`。

### pass4WeeklyRest — 无需改动
`'休(补)'` 的 `type !== '白'`，现有 `cell.type !== '白'` 条件已自动跳过，pass4 不会覆盖该格子，且 `_calcRestScore` 已正确统计 0.5 天贡献。

## 行为变化对比

| 场景 | 旧行为 | 新行为 |
|------|--------|--------|
| 2 人白班池备班次日 | `'补'`（0 天配额，需靠其他 5 天补满 1.5 天）| `'休(补)'`（0.5 天配额，其余 5 天只需补 1.0 天）|
| 3+ 人白班池备班次日 | `'补'` | `'补'`（不变）|
| 每周配额统计 | `'休(补)'` 未知类型按 0 算 | 正确计 0.5 天 |
| 图例显示 | 无 `'休(补)'` 条目 | 已添加说明 |
