# 排班规则升级 v2 总结

## 核心变更

### schedule.js 完整重写

**班次配置**
- 删除 `长` 班次（护士长改为显示 `白`/`休`）
- 新增 `上`（上午班，0.5天，黄色）和 `午`（下午班，0.5天，绿色）
- `STAFF_LIST` 中护士长增加 `badge: '长'`，备孕人员增加 `badge: '孕'`

**四遍排班算法**

| Pass | 职责 |
|------|------|
| Pass 1 | 基础班次：倒班按主→夜→下循环；头/白班/备孕全部默认 `白` |
| Pass 2 | 每天从候选池 `[r_a, r_b, r_c, d_a, d_b, d_c]` 轮转指派备班；备班次日设 `午` |
| Pass 3 | 白班/备孕人员的 `休` 次日设 `午` |
| Pass 4 | 逐人逐周校验休息量（`休`=1, `下`=1, `午`/`上`=0.5），不足 2 天则在轮休日补 `休`，同时触发 Pass3 逻辑 |

**护士长规则更新**：全天默认 `白`，参与 Pass4 每周补 2 天休息（在轮休日自动分配）

### schedule.wxml
- 名称列新增 badge 角标：`person.badge` 存在时渲染 `<view class="name-badge">`
- 规则说明卡片文案更新（新增备班次日午班、白班一天半、每周2天休息规则）

### schedule.wxss
- 新增 `.name-badge` 样式（小圆角标签，颜色继承人员色）

## 关键约束保证

- 每天至少 1 人备班（兜底选主班倒班人员）
- 轮休日至少 1 名白班人员在岗（Pass4 补休前校验）
- 手动编辑（`manual=true`）的格子全程不被任何 Pass 覆盖
