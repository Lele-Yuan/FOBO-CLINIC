# 排班规则升级 v2 文档

## 一、变更需求汇总

| 编号 | 新规则 | 当前实现 | 变化 |
|------|--------|----------|------|
| R1 | 每天有且只有一个备班 | 仅工作日、仅白班人员轮备 | 扩展至每天，包含倒班人员 |
| R2 | 所有班次都可备班 | 仅 role=day 参与备班 | 倒班（主/夜）也可备班；休息中（下/休）不备班 |
| R3 | 备班第二天上午必须休（半天） | 无此规则 | 备班次日自动标记为 `午` |
| R4 | 白班休息一天半 | 休息日后直接上班 | 白班人员 `休` 次日自动标记为 `午` |
| R5 | 护士长/备孕名称处标注，班次不体现 | 护士长有专用 `长` 班次 | 删除 `长`，名称列增加角标 |
| R6 | 护士长也遵循轮休规则（周二/四/六/日可休） | 护士长每天上白班无休 | 护士长加入轮休池 |
| R7 | **所有人每周必须有 2 天休息**（`午`/`上` 计 0.5 天） | 无强制校验 | 生成后逐人逐周校验，不足 2 天则在轮休日补 `休` |

## 二、新增/删除班次类型

| 班次 | 含义 | 触发场景 | 背景色 | 文字色 |
|------|------|----------|--------|--------|
| `上` | 上午班（仅上午，下午休，计 0.5 天） | 预留，可手动指定 | `#FFF3CD` | `#856404` |
| `午` | 下午班（仅下午，上午休，计 0.5 天） | 备班次日 / 休息次日 | `#D4F0E0` | `#3A8A5C` |

删除班次：`长`（原护士长专属，改为显示 `白`/`休`）

> **半天计算规则**：每周休息量 = Σ(`休`×1) + Σ(`午`×0.5) + Σ(`上`×0.5) + 倒班`下`×1，必须 ≥ 2.0

## 三、算法设计（四遍生成）

### Pass 1 — 基础班次

- **护士长**：工作日（周一/三/五）→ `白`；轮休日（周二/四/六/日）→ 参与轮休排班（默认 `白`，由 Pass4 补足休息）
- **倒班三人**：按原 主→夜→下 循环，不受轮休日影响
- **白班三人**：周一/三/五 → `白`；轮休日 → 默认 `白`（由 Pass4 补足休息）
- **备孕**：同白班三人

### Pass 2 — 备班分配

**备班候选池**（每天动态计算）：
```
候选 = 全体人员 中排除：
  - 护士长（不参加备班）
  - 备孕人员（不参加备班）
  - 当天已是 `下` 或 `休` 的人
```

**轮转策略**：维护全局 `standbyPointer`，候选顺序 `[r_a, r_b, r_c, d_a, d_b, d_c]`，每天顺序取下一可用人：

1. 将选中人当天班次覆盖为 `备`
2. 将选中人**次日**班次覆盖为 `午`（不覆盖 `备` 和 `manual=true`）

兜底：候选池为空时，选当天上 `主` 班的倒班人员。

### Pass 3 — 白班休息半天

遍历所有白班/备孕人员（role=day / day_prep），若某天为 `休`：
- 次日设为 `午`（不覆盖 `备` 和 `manual=true`）

### Pass 4 — 每周 2 天休息强制校验

对每个人，**按自然周（周一～周日）**统计当周休息量：

```
restScore = count(休)*1 + count(下)*1 + count(午)*0.5 + count(上)*0.5
```

若 `restScore < 2`，在该周**轮休日（周二/四/六/日）**中选尚未安排休息的天，补充 `休`，直到 `restScore ≥ 2`。

补休优先级：
1. 优先选已有最多人在岗、不影响"至少一人在岗"约束的日期
2. 同优先级取最早的轮休日
3. 不在 `manual=true` 的格子上补休

## 六、边界条件

| 场景 | 处理 |
|------|------|
| 月末备班，次月第一天需标 `午` | 生成时向后多取 1 天，`午` 写入后裁剪回当月 |
| 当天候选池全空 | 选当天 `主` 班倒班人员兜底 |
| `午` 次日又是 `备` | `备` 优先，不覆盖 |
| Pass4 补休后导致某天无人在岗 | 跳过该天，选下一轮休日 |
| 手动修改格子 | `manual=true` 全程不被任何 Pass 覆盖 |
| 倒班 `下` 落在工作日（周一/三/五） | 倒班性质特殊，`下` 不受轮休日限制，直接计入休息量 |
